대규모 트래픽 광고 서버 부하 테스트 및 성능 개선

KT 나스미디어

실시간 광고 처리 최적화 및 안정성 검증

C/Linux HTTP/epoll k6 / hey

Problem & Goal

대규모 실시간 트래픽 환경에서 서버의 병목 구간을 파악하고, 운영 환경에 배포하기 전 최대 수용 가능한 성능 지표를 확보해야 했습니다.

Implementation

  • k6와 hey를 활용해 최대 490만 건 요청, 수만 RPS 규모의 스트레스 테스트(Stress Test) 시나리오 설계 및 수행
  • Socket 통신 단계의 병목을 분석하고, HTTP Keep-Alive 설정 및 epoll 기반의 네트워크 I/O 구조 최적화

Result

측정 API의 경우 22,000 RPS 환경에서 평균 응답속도 66~210ms를 유지하며 무결점(Error 0) 처리 성능을 확인했습니다. 광고 요청(AdTag) 처리 시에는 목표 트래픽을 상회하는 최대 82,000 RPS의 극단적 부하 상황에서도 490만 건 이상의 요청을 에러 없이(100% 성공) 소화해내며 대규모 운영 서버 배포의 신뢰성을 완벽히 입증했습니다.
